Heroic Development — UX Designer (2024–2025)

Heroic Development builds board management software for organizational governance. I joined as a summer intern in 2023 and returned as a full designer during my senior capstone and after graduation.

What I worked on

  • Dashboard and navigation — information architecture and main navigation design across the core product
  • Meeting manager — end-to-end flow for scheduling, running, and documenting board meetings
  • Document editor — in-app editing with real-time collaboration indicators and version history
  • Member manager — admin interface for role assignment and permission management
  • Login and authentication — sign-in, password reset, and first-time-setup flows
  • Responsive design — all features for both desktop and mobile-optimized layouts

Process

Each feature began with a written workflow document — I mapped user flows and edge cases in writing before opening Figma. Then: wireframes, design review, high-fidelity mockups, interactive prototype for developer handoff. Weekly design reviews with the product team throughout.

Impact

  • Designed and delivered wireframes and prototypes for 6+ major feature areas from concept to developer handoff.
  • Competitive analysis across 4 comparable SaaS platforms to benchmark usability and inform decisions.
  • High-fidelity prototypes used directly in investor and stakeholder presentations — received positive feedback.

Supergreen — UX / Product Designer (2025–2026)

At Supergreen I worked across multiple concurrent SaaS products, contributing to a shared design system and individual product interfaces.

What I worked on

  • Design system application across multiple products simultaneously
  • Light and dark mode — all interfaces for both modes, maintaining contrast and hierarchy across themes
  • Messaging and notifications — real-time messaging UI, notification center, alert patterns
  • Directory and resource features — searchable directory and resource library UIs
  • Feature specification — requirement documents before wireframing to align scope
  • Rapid iteration — weekly review cycles, updating designs in tight turnaround windows

Key Learnings

What professional practice taught me

  • Speed and quality require good systems, not a trade-off. Design systems were the key — a solid component library meant I could move fast without sacrificing consistency.
  • Writing before wireframing saved time, not lost it. Feature requirement docs forced scope clarity, surfaced edge cases early, and made developer conversations more productive. I now do this on every project.
  • Feedback is the product. A design isn't finished when you think it's good — it's finished when tested by the team, reviewed by stakeholders, and validated against what developers can build. Learning to act on feedback quickly without losing quality is a real skill.
